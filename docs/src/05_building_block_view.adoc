== Building Block View

=== Whitebox Overall System

[plantuml, target=level1, format=png]
....
@startuml
!define RECTANGLE_STYLE
skinparam linetype ortho
skinparam shadowing false
skinparam roundcorner 5

title Gamey Ecosystem - Context (Level 1)

actor "Client / Player" as Client
rectangle "Gamey System" as System #D3D3D3
rectangle "Firebase" as Firebase #F2F2F2

Client -right-> System : Uses / Plays
System -right-> Firebase : Auth & Persistence
@enduml
....

Motivation::
The system uses a hybrid backend approach. **Firebase** is used as a Third-Party Building Block to offload identity management and persistence. This allows the **Gamey system** to focus on custom business logic and the handle CPU-intensive game mechanics without managing user sessions directly.

Contained Building Blocks::
[cols="1,2" options="header"]
|===
| **Name** | **Responsibility**
| **Gamey System** | Custom and developed layers of the application with orchestrated services.
| **Firebase** | External black box providing Authentication and cloud-based data storage.
|===

==== Gamey System
Purpose/Responsibility::
Coordinates the user experience and provides a single gateway that communicates the backend and frontend interfaces.

==== Firebase
Purpose/Responsibility::
Provides a secure and external layer for handling the Data Base.

==== Game Engine
Purpose/Responsibility::
The computational core. It is agnostic of user identity (delegated to other blocks) and focuses on the game rules.

' Internal Routing (Behind Gateway)

GameMgr -down-> Redis : Cache State

Motivation::
The **Gamey System** divided so that it has an only entry point and (not counting the Redis cache) there is no direct connection among the internal elements and only. As so there is only 1 frontend element (**WebApp**) whilst there exists 3 backend elements, one service (**Gamey**) and two microservices (**Auth** and **Game Manager**).

Contained Building Blocks::
[cols="1,2" options="header"]
|===

| **WebApp** | Frontend of the application.
| **Auth Engine** | Microservice with the purpose of communicating with the database to ensure.
| **Game Manager** | Custom and developed layers of the application with orchestrated services.
| **Gamey** | Custom and developed layers of the application with orchestrated services.
| **Redis** | Custom and developed layers of the application with orchestrated services.
| **Firebase** | External black box providing Authentication and cloud-based data storage.
|===
